# 📋 AI-Code 面试准备清单

基于 AI-Code 项目的完整面试准备指南，涵盖技术深度、项目经验、系统设计等各个方面。

## 🎯 项目概览准备

### 📝 Blog 应用 (Next.js 15 全栈博客)
**核心亮点**: AI 集成 + 现代化全栈架构 + 性能优化

#### 必准备内容
- [ ] **技术栈掌握**: Next.js 15、React 18、TypeScript、Prisma 6.11
- [ ] **AI 集成**: OpenAI GPT 集成、SSE 流式响应实现
- [ ] **数据库设计**: PostgreSQL + Prisma ORM + 查询优化
- [ ] **认证系统**: NextAuth.js + JWT + 权限管理
- [ ] **性能优化**: SSR、代码分割、缓存策略、Lighthouse 95+

#### 面试问题准备
1. **Q: 如何实现 AI 聊天的实时流式响应？**
   - A: 使用 SSE (Server-Sent Events) 技术，结合 OpenAI 的流式 API
   - 关键技术：ReadableStream、TextEncoder、错误处理机制
   
2. **Q: Next.js 15 相比之前版本有什么优势？**
   - A: 更好的性能、改进的 App Router、增强的类型安全
   - 具体体现在：更快的构建速度、更好的开发体验

3. **Q: 如何优化 Markdown 渲染的性能？**
   - A: 虚拟滚动、懒加载、DOMPurify 安全防护、语法高亮优化

### 📱 Mobile 应用 (三端统一 BFF 架构)
**核心亮点**: BFF 架构 + 自定义 SSR + 智能数据裁剪

#### 必准备内容
- [ ] **BFF 架构**: Backend for Frontend 设计理念和实现
- [ ] **客户端适配**: 智能识别客户端类型并优化数据响应
- [ ] **自定义 SSR**: 不依赖 Next.js 的服务端渲染实现
- [ ] **性能优化**: 首屏渲染 < 800ms、数据传输优化 30-50%

#### 面试问题准备
1. **Q: 什么是 BFF 架构？为什么选择这种架构？**
   - A: BFF 是为前端定制的后端服务，可以针对不同客户端优化数据
   - 优势：数据裁剪、性能优化、客户端适配、开发效率提升

2. **Q: 如何实现三端统一的数据服务？**
   - A: 通过客户端识别 + 数据适配器模式，根据客户端类型自动优化数据

3. **Q: 自定义 SSR 相比 Next.js SSR 有什么优势？**
   - A: 更灵活的控制、更好的性能、更小的包体积、完全自定义的渲染逻辑
